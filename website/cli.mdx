Run `uv run volo <command> --help` for full flags.

## `volo init`
Quickstart: record an agent and score it in one step.
```bash
uv run volo init <module:fn> --input '{...}' [--name baseline] [--n 2]
```

## `volo record`
Capture a live agent run as a Recording.
```bash
uv run volo record <module:fn> --input '{...}' --out ./.volo/recordings/run.json
```

## `volo sim`
Boot the simulated environment from a recording and print the transcript (deterministic replay).
```bash
uv run volo sim ./.volo/recordings/run.json
```

## `volo run`
Run an agent against scenarios; produce a ReliabilityReport + verdict.
```bash
uv run volo run <recording> --agent <module:fn> --input '{...}' \
  [--n 3] [--fail-under 0.9] [--judge heuristic|ollama|groq] [--max-cost-usd 0.50] [--out report.json]
```

## `volo ci`
PR-gating entrypoint: same engine as `run`, CI-friendly output, non-zero exit on regression.
See [Gate pull requests in CI](/ci). Adds `--summary-md PATH`.

## `volo comment`
Render a sticky PR-comment Markdown from a reliability report (+ optional compliance evidence).
Used by the [PR-check Action](/pr-check); emits UTF-8 to stdout / `--out`.
```bash
uv run volo comment --report report.json [--evidence evidence.json] [--out comment.md]
```

## `volo diff`
Diff two runs and attribute the regression to a step ("git bisect for agents").
```bash
uv run volo diff <baseline.json> <candidate.json>
```

## `volo mcp record`
Transparent proxy between an MCP client and the real server; saves the session as a Recording.
stdout stays pure protocol — diagnostics go to stderr.
```bash
uv run volo mcp record --out rec.json [--server-name NAME] -- <server command...>
```

## `volo mcp serve`
Simulated MCP server replayed from a recording — offline, byte-identical, un-recorded inputs
return JSON-RPC error `-32042` instead of a hallucination. See [Test MCP servers](/mcp).
```bash
uv run volo mcp serve rec.json
```

## `volo mcp fuzz`
Mutate recorded tool responses with the adversarial scenario operators (seeded, reproducible);
write one servable mutated recording per operator, or serve one hostile world directly.
```bash
uv run volo mcp fuzz rec.json [--seed 0] [--out-dir DIR] [--report fuzz.json]
uv run volo mcp fuzz rec.json --op prompt_injection --serve
```

## `volo mcp conformance`
Replay recorded requests against the LIVE server and diff the answers (recorded errors included).
Exit 1 if behavior changed — a CI regression gate for MCP-server authors.
```bash
uv run volo mcp conformance rec.json [--report conf.json] -- <server command...>
```

## `volo shadow pull` / `adopt` / `list`
Bank production OTel traces (redacted before disk, deduplicated by content) or adopt a failure
trace as a permanent regression. See [Production shadow](/shadow).
```bash
uv run volo shadow pull traces/ [--corpus ./.volo/corpus] [--agent-name NAME]
uv run volo shadow adopt failure.json [--tag incident]
uv run volo shadow list
```

## `volo shadow check`
Replay the whole corpus against the current agent build; compare the reliability surface to the
last snapshot. Exit 0 = no drift, **exit 3 = drift alert** (dimension dropped > threshold or a
verdict flipped). First run establishes the baseline. Every check appends to the trend history;
`--webhook` (or `VOLO_SHADOW_WEBHOOK`) POSTs a Slack-compatible alert on drift.
```bash
uv run volo shadow check --agent <module:fn> [--threshold 0.05] [--report drift.json] \
  [--update-baseline] [--history ./.volo/shadow-history.jsonl] [--webhook URL]
```

## `volo shadow trend`
Reliability-over-time from the check history: one ASCII sparkline per dimension, fleet average
by default, `--trace <run_id>` for a single banked trace. Feeds the `/shadow` dashboard screen.
```bash
uv run volo shadow trend [--history PATH] [--trace RUN_ID]
```

## `volo redteam run` / `list` / `export`
Run the adversarial attack corpus against an agent in the sim; exit 4 if any attack lands.
`list` shows the corpus by class; `export` writes it as an editable JSON pack. See
[Red-team your agent](/redteam).
```bash
uv run volo redteam run <recording> --agent <module:fn> [--input '{...}'] [--pack pack.json] [--out annex.json]
uv run volo redteam list [--pack pack.json]
uv run volo redteam export my-pack.json
```

## `volo migrate`
Compare an agent's reliability + cost across two models: pair a baseline corpus (model A) with a
re-recorded candidate corpus (model B), score each pair, recommend go/review/block. Exit 5 on
block. See [Model-migration lab](/migrate).
```bash
uv run volo migrate <baseline> <candidate> [--from LABEL] [--to LABEL] [--judge heuristic|ollama|groq] [--out report.json]
```

## `volo persona run` / `list` / `export`
Drive a multi-turn agent against a seeded persona (simulated user/counterparty); print the
conversation + goal verdict. `--require-goal` exits 6 if the agent misses the goal. See
[Simulated users](/personas).
```bash
uv run volo persona run --agent <module:fn> --persona <name|pack.json> [--recording rec.json] [--require-goal] [--out conv.json]
uv run volo persona list
uv run volo persona export <name> out.json
```

## `volo horizon`
Long-horizon drift rig: replay a task for N episodes with memory threaded forward; report how
reliability decays. Exit 7 on `degrades`. See [Long-horizon rig](/horizon).
```bash
uv run volo horizon <recording> --agent <module:fn> [-n 20] [--judge heuristic|ollama|groq] [--out report.json]
```

## `volo pack init` / `validate` / `install` / `list`
Build, verify, and install versioned content packs (attacks / personas / scenarios). `validate`
checks manifest + checksum + item schema (exit 1 on failure). See [Content packs](/packs).
```bash
uv run volo pack init <kind> out.json [--name N] [--version 1.0.0]
uv run volo pack validate pack.json
uv run volo pack install pack.json [--dir ./.volo/packs] [--force]
uv run volo pack list
```

## `volo pack publish` / `install --registry` / `search`
Publish a pack to a git-backed registry index and install packs by name (checksum-verified).
No registry service — the index is a JSON file in a repo. See [Content packs](/packs).
```bash
uv run volo pack publish pack.json --url <hosted-url> --index index.json
uv run volo pack search --registry <index-url-or-path> [name]
uv run volo pack install <name> --registry <index-url-or-path> [--version X] [--keyring k.json] [--require-signed]
```

## `volo pack sign` / `verify`
Sign a pack with a publisher secret (HMAC-SHA256) so installers can verify provenance; the
registry marks signed versions `verified`. `verify` exits 1 on a bad/absent signature.
```bash
export VOLO_PACK_SECRET=...   # the publisher's secret
uv run volo pack sign pack.json --publisher <name>
uv run volo pack verify pack.json --keyring keyring.json
```

## `volo compliance build` / `verify`
Bundle reliability / red-team / drift reports into a signed, deterministic evidence pack mapped to
EU AI Act / ISO 42001 / SOC 2 controls; `--require-satisfied` exits 8 on any unmet control.
`verify` checks the checksum (and signature with `--keyring`). See [Compliance](/compliance).
```bash
uv run volo compliance build --agent NAME [--reliability r.json] [--safety a.json] [--drift d.json] \
  [--framework all] [--out evidence.json] [--md report.md] [--sign PUBLISHER] [--require-satisfied]
uv run volo compliance verify evidence.json [--keyring keyring.json]
```

## `volo cu inspect` / `replay`
Inspect or replay a computer-use recording (browser/desktop agents) at the action-event level.
`replay` serves recorded UI outcomes for action events read from stdin; unseen (action, screen)
flags. See [Computer-use](/computeruse).
```bash
uv run volo cu inspect <recording>
uv run volo cu replay <recording>   # JSON action per line on stdin
```

## `volo scenarios`
List the default adversarial scenario operators and their failure classes.

## `volo demo`
Seed the data dir with showcase recordings + reports for the dashboard.
